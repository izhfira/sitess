    Mengukur Jarak Data

    A. Mengukur Jarak Tipe Data Numerik

    1. Minkowski Distance

    Minkowski distance adalah jarak metrik yg berada dalam ruang vektor yang dapat dianggap sebagai generalisasi dari jarak Euclidean dan jarak Manhattan.

    Berikut cara untuk menghitung jarak Minkowski : $$ d _ { \operatorname { min } } = ( \sum _ { i = 1 } ^ { n } | x _ { i } - y _ { i } | ^ { m } ) ^ { \frac { 1 } { m } } , m \geq 1 $$ Keterangan :

    n = banyak fitur

    i = batas jumlah data ke - i

    m = banyaknya data

    xi = nilai x pada data ke - i

    yi = nilai y pada data ke - i

    1. Manhattan Distance

    Manhattan distance adalah jarak untuk menentukan suatu titik menuju titik lainnya di bidang cartesian dengan melewati bagian vertikal dan horizontal tanpa pernah kembali.

    Berikut cara untuk menghitung jarak Manhattan : $$ d _ { \operatorname { man } } = \sum _ { i = 1 } ^ { n } \left| x _ { i } - y _ { i } \right| $$ Keterangan :

    n = banyak fitur

    i = batas jumlah data ke - i

    m = banyaknya data

    x dan y = sebuah titik yang akan diukur jaraknya

    1. Euclidean Distance

    Euclidean distance adalah jarak yg diukur lurus dari fasilitas titik yang satu ke fasilitas titik yang lain dalam Euclidean space. Metode perhitungan euclidean distance bisa digunakan untuk menghitung jarak terdekat.

    Berikut cara untuk menghitung jarak Euclidean :

    n = banyak fitur

    i = batas jumlah data ke - i

    m = banyaknya data

    xik = nilai x pada training data

    yik = nilai y pada training data

    1. Average Distance

    Average distance jarak yang merupakan salah satu bentuk modifikasi dari perhitungan jarak Euclidean yang berguna untuk memperbaiki hasil yang didapatkan. Dua titik x,y merupakan titik dalam sebuah ruang dimensi -n.

    Berikut rumus untuk mendefinisikan Average distance : $$ d _ { a v e } = \left ( \frac { 1 } { n } \sum _ { i = 1 } ^ { n } ( x _ { i } - y _ { i } ) ^ { 2 } \right) ^ { \frac { 1 } { 2 } } $$ Keterangan :

    n = banyak fitur

    i = batas jumlah data ke - i

    m = banyaknya data

    xik = nilai x pada training data

    yik = nilai y pada training data

    1. Weighted Euclidean Distance

    Weighted Euclidean distance adalah jarak yang merupakan modifikasi lain dari Euclidean distance yg berguna untuk meminimalkan hasil yang didapatkan agar lebih sesuai.

    Berikut rumus dari jarak Weighted Euclidean : $$ d _ { w e } = \left ( \sum _ { i = 1 } ^ { n } w _ { i } ( x _ { i } - y _ { i } \right) ^ { 2 } ) ^ { \frac { 1 } { 2 } } $$

    1. Chord Distance

    Chord distance adalah salah satu ukuran jarak modifikasi dari Euclidean distance yang digunakan untuk memperbaiki kelemahan dari Euclidean distance supaya hasil yang didaptkan lebih maksimal dan sesuai dengan yang diinginkan. Perhitungan Chord distance dapat diselesaikan juga dengan menggunakan skala pengukuran yang baik dan bisa dihitung dari data yang tidak dinormalisasi.

    Berikut rumus untuk menghitung jarak Chord : $$ d _ { \text {chord} } = \left ( 2 - 2 \frac { \sum _ { i = 1 } ^ { n } x _ { i } y _ { i } } { | x | _ { 2 } | y | _ { 2 } } \right) ^ { \frac { 1 } { 2 } } $$

    1. Mahalanobis Distance

    Mahalanobis distance merupakan suatu metode statistika yang dipergunakan untuk mendapatkan suatu data dengan jarak tertentu terhadap mean data tersebut sehinggan diperoleh suatu penyebaran data yang memiliki pola terhadap nilai mean. Mahalanobis distance dipergunakan dalam Digital Image Processing untuk mengidentifikasi variant selama pengklasifiksian gambar. Mahalanobis distance adalah jarak yang berdasarkan dara berbeda dengan Euclidean distance dan Manhattan distance yang bebas antara data yang satu dengan data yang lain.

    Berikut rumus untuk menghitung Mahalanobis distance : $$ d _ { m a h } = \sqrt { ( x - y ) S ^ { - 1 } ( x - y ) ^ { T } } $$

    1. Cosine Measure

    Ukuran Cosine similarity lebih dipergunakan dalam similaritas dokumen.

    Berikut rumus untuk menyatakn Cosine similarity : $$ Cosine(x,y)=\frac { \sum _ { i = 1 } ^ { n } x _ { i } y _ { i } } { | x | _ { 2 } | y | _ { 2 } } $$

    1. Pearson Correlation

    Pearson correlation adalah salah satu ukuran korelasi yang berguna untuk mengukur kekuatan dan arah hubungan suatu linear dari dua variabel. Dua variabel berkorelasi jika perubahan salah satu variabel disertai dengan perubahan variabel yang lain baik berasal dari arah yang sama maupun bukan.

    Berikut cara untuk menghitung korelasi Pearson : $$ Pearson ( x , y ) = \frac { \sum _ { i = 1 } ^ { n } ( x _ { i } - \mu _ { x } ) ( y _ { i } - \mu _ { y } ) } { \sqrt { \sum _ { i = 1 } ^ { n } ( x _ { i } - y _ { i } ) ^ { 2 } } \sqrt { \sum _ { i = 1 } ^ { n } ( x _ { i } - y _ { i } ) ^ { 2 } } } $$

    B. Mengukur Jarak Atribut Binary

    Atribut Binary adalah nominal yang hanya berisikan dua jenis nilai saja, yaitu 0 dan 1. Yang mana 0 berarti tidak ada dan 1 berarti ada. Atribut binary biasanya diartikan sebagai boolean jika kedua keadaannya bernilai benar (true) dan salah (false). Atribut binary bisa berbentuk simetris ataupun asimetris. Bentuk simetris jika kedua nilainya bernilai sama, sehingga tidak bisa diberi kode 0 atau 1. Sedangkan bentuk asimetris jika kedua nilainya tidak seimbang, sehingga bisa diberi kode 0 atau 1.

    Berikut rumus untuk atribut biner simetris : $$ d ( i , j ) = \frac { r + s } { q + r + s + t } $$ Berikut rumus untuk atribut biner asimetris : $$ d ( i , j ) = \frac { r + s } { q + r + s } $$ Pengukuran perbedaan antara dua atribut biner berdasarkan pada disimilarity. Misalnya, biner asimetris kesamaan antara objek i dan j dapat dihitung dengan rumus : $$ \operatorname { sim } ( i , j ) = \frac { q } { q + r + s } = 1 - d ( i , j ) $$ C. Mengukur Jarak Tipe Data Categorical

    1. Overlay Metric

      Overlay metric biasanya dipergunakan oleh instance-based learning dan locally weighted learning. Hal ini bisa dilihat apabila overlay metric tidak efisien dalam mengukur jarak antara masing-masing pasangan sample, karena gagal dalam memanfaatkan tambahan informasi yang diberikan oleh nilai atribut nominal yang bisa membantu dalam generalisasi. $$ d ( x , y ) = \sum _ { i = 1 } ^ { n } \delta ( a _ { i } ( x ) , a _ { i } ( y ) ) $$

    2. Value Difference Metric

      Value difference metric diperkenalkan oleh Standfill dan Waltz, versi sederahana dari value difference metric tanpa skema pembobotan dana didefinisikan dengan : $$ d ( x , y ) = \sum _ { i = 1 } ^ { n } \sum _ { c = 1 } ^ { C } \left| P ( c | a _ { i } ( x ) ) - P ( c | a _ { i } ( y ) ) \right | $$ Value difference metric mengasumsikan bahwa dua nilai dari atribut adalah lebih dekat jika memiliki klasifikasi yang sama. Pendekatan lain yang berbasis probabilitas adalah Short Fukunaga Metric (SFM) yang kemudian dikembangkan oleh Myles dan Hand dan didefinisikan dengan : $$ d ( x , y ) = \sum _ { c = 1 } ^ { C } \left | P ( c | x ) - P ( c | y ) \right| $$

    3. Minimum Risk Metric

      Minimum risk metric adalah ukurang yang dipresentasikan oleh Blanzieri dan Ricci. Minimum risk metric meminimalkan resiko kesalahan dalam pengklasifikasian suatu sample data yang dapat didefinisikan dengan rumus berikut : $$ d ( x , y ) = \sum _ { c = 1 } ^ { C } P ( c | x ) ( 1 - P ( c | y ) ) $$

    D. Mengukur Jarak Tipe Ordinal

    Atribut ordinal adalah atribut yang nilai-nilainya berkemungkinan memiliki urutan yang memiliki arti atau tingkatan (ranking). Akan tetapi jarak diantara nilai-nilai tersebut tidak diketahui. Atribut ordinal bergunu untuk mendaftarkan taksiran suatu kualitas yang tidak bisa diukur secara obyektif. Oleh sebab itu atribut ordinal biasanya digunakan dalam survey atau rating. dikarenakan setiap atribut ordinal memiliki jumlah state yang berbeda, diperlukan untuk memetakan rentang setiap atribut [0,0,1.0] sehingga setiap atribut memiliki bobot yang sama dengan menggunakan rumus berikut : $$ z _ { i f } = \frac { r _ { i f } - 1 } { M _ { f } - 1 } $$ E. Menghitung Jarak Tipe Campuran

    Menghitung ketidaksamaan antara objek dengan atribut campuran yang berupa nominal, biner simetris, biner asimetris, numerik, atau ordinal yang ada pada kebanyakan database dapat dinyatakan dengan memproses semua tipe atribut secara bersamaan. salah satu teknik tersebut yaitu menggabungkan atribut yang berbeda ke dalam matriks ketidaksamaan tunggal dan menyatakannya dengan skala interval antar [0,0,1.0].

    Misalkan data berisi atribut p tipe campuran. Ketidaksaaan (disimilarity) anatara objek i dan j dinyatalan dengan : $$ d ( i , j ) = \frac { \sum _ { f = 1 } ^ { p } \delta _ { i j } ^ { ( f ) } d _ { i j } ^ { ( f ) } } { \sum _ { f = 1 } ^ { p } \delta _ { i j } ^ { ( f ) } } $$ F.     #Menghitung Jarak Numerik
def chordDist(v1,v2,jnis):
    jmlh=0
    normv1=0
    normv2=0
    for x in range (len(jnis)):
        normv1=normv1+(int(k.values.tolist()[v1][jnis[x]])**2)
        normv2=normv2+(int(k.values.tolist()[v1][jnis[x]])**2)
        jmlh=jmlh+(int(k.values.tolist()[v1][jnis[x]])*int(k.values.tolist()[v2][jnis[x]]))
    return ((2-(2*jmlh/((normv1**0.5)*(normv2**0.5)))**0.5))

    #Menghitung Jarak Binary
def binaryDist(v1,v2,jnis):
    q=0
    r=0
    s=0
    t=0
    for x in range (len(jnis)):
        if (int(k.values.tolist()[v1][jnis[x]]))==1 and (int(k.values.tolist()[v2][jnis[x]]))==1:
            q=q+1
        elif (int(k.values.tolist()[v1][jnis[x]]))==1 and (int(k.values.tolist()[v2][jnis[x]]))==2:
            r=r+1
        elif (int(k.values.tolist()[v1][jnis[x]]))==2 and (int(k.values.tolist()[v2][jnis[x]]))==1:
            s=s+1
        else:
            t=t+1
    return ((r+s)/(q+r+s+t))
